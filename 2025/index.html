演奏間違いを検出して楽譜上に指摘の表示をしてみる

13:50 〜 17:50

中級者向けAIApplicationUI/UX音楽情報処理

初心者は楽器練習中に自分の演奏が正しいか間違いか分からないことがあります。 指導者がいれば間違いを指摘できます。しかし、自宅練習では指摘できる人がおらず間違った練習を続け、上達の妨げになることもあります。 そこで、演奏間違いを検出してどのような間違いがあったか楽譜上で指摘する表示をするツールを作ってみました。

指摘する方法として、技術的にはピアノロール表示を応用するのが簡単そうです。 とはいえ、ピアノロールは DTM に親しんでいる方なら容易に理解できそうな一方で、一般的とは言えず理解が難しい方も多数います。 一方、ピアノなど鍵盤楽器においては五線譜という楽譜が共通言語であり、初心者にも楽譜が読めるようになるための指導がされます。 そこで、楽譜の上にどのような間違いをしたかの指摘を重ねて表示することで、初心者本人だけでなく指導者や保護者にとっても共通言語により理解しやすいようにしています。

また、楽譜上に指摘表示するためには、すべての音符がそれぞれどの位置に表示されているかの情報が必要になりますが、手作業でいちいち指定していくのは大変です。 そこで、楽譜作成プログラム LilyPond を使い、表示用の楽譜と位置情報の両方を同時に自動生成することにより省力化が図れるようにしています。 今回は鍵盤楽器を用意してご来場された方にご試用いただきます。

（一時的に展示を中断する場合があります）

学べること

  • 音楽情報処理における1つの技術

参考文献

このような実は身近に存在する CRIU を使ったチェックポイント／レストア機能、使ったことありますか？

使うとどんないいことがあるのでしょうか？そもそもどこで・どのように使えばいいのでしょうか？それらについてサンプルコードを交えながらご紹介します。

学べること

  • CRIU の特徴、採用のメリット

NetBox をオンプレから AWS へ移してみた

16:00 〜 16:30

中級者向けDevOpsDatabaseContainerCloudApplication

NetBox という IPAM や DCIM を実施するためのアプリケーションがあります。

私達は日々この NetBox を利用してリソースの管理を実施しています。

以前はこの NetBox をオンプレミスのサーバー上で Docker Compose を利用していて管理していました。

しかし、オンプレミスのサーバーが法定停電を実施しなければならない場所に設置していたため、法定停電があると NetBox 上のリソースが見えなくなるという問題がありました。

また、冗長構成やバックアップが考慮していなかったためいつでもデータロストの危険性やサービスの継続不可能性を孕んでいました。

そのため、冗長性やバックアップ性能を確保しつつ常に NetBox をサービスとして稼動させるため、AWS へ移行を実施しました。

その際に得られた知見や冗長性やバックアップアーキテクチャ、運用方法等について共有します。

学べること

  • オンプレミスのサービスを AWS へ移行するノウハウや AWS 上での Web サービスの構築におけるベストプラクティス
